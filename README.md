\# Crypto Fee Router\## Preview!\[Crypto Fee Router Dashboard](assets/dashboard-preview.png)\## Demo\[Watch demo video](assets/demo.webm)A lightweight multi-chain fee routing dashboard that compares transaction costs across EVM networks and recommends the best route based on cost, speed, reliability, and risk.\## What it does\- Fetches live fee data from multiple EVM chains\- Compares Ethereum, Base, Arbitrum, Optimism, and Polygon\- Estimates transaction costs\- Calculates a routing score for each chain\- Displays the results in a real-time Streamlit dashboard\- Uses a FastAPI backend for fee aggregation and routing logic\## Tech Stack\- Python\- FastAPI\- Web3.py\- Streamlit\- Pandas\- Uvicorn\## Project Structure```textcrypto-fee-router/├── app/│   ├── api/│   ├── core/│   ├── models/│   └── services/├── assets/│   ├── dashboard-preview.png│   └── demo.webm├── dashboard.py├── requirements.txt├── .env.example└── README.md```\## Setup```bashgit clone https://github.com/yakirinfra/crypto-fee-router.gitcd crypto-fee-routerpython -m venv venvvenv\\Scripts\\activatepip install -r requirements.txt```Create a `.env` file based on `.env.example` and add your RPC URLs.\## Run the API```bashuvicorn app.main:app --reload```API docs:```texthttp://127.0.0.1:8000/docs```\## Run the DashboardOpen a second terminal:```bashstreamlit run dashboard.py```\## PurposeThis project demonstrates practical Web3 infrastructure skills: multi-chain RPC usage, fee monitoring, backend API design, and real-time dashboard visualization.\## DisclaimerThis project is for educational and portfolio purposes only.
